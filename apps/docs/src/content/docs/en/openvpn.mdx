Daytona provides an option to run sandboxes that can be connected to private networks. One common use case is accessing the sandbox over a private mesh network using [OpenVPN](https://openvpn.net/).

## 1. Setup

### Daytona account

Open [Daytona Dashboard â†—](https://app.daytona.io/) to create or access your Daytona account.

### Daytona API key

Generate an [API key](/docs/en/api-keys).

### Daytona sandbox

Create a Daytona sandbox to connect to OpenVPN network. The following snippet is an example of creating a default sandbox using the [Daytona SDK](/docs/en/getting-started#sdks). For more information on creating and configuring a Daytona sandbox, see the [sandboxes documentation](/docs/en/sandboxes).

### OpenVPN account

Create or access your [OpenVPN account](https://openvpn.net/).

## 2. Install OpenVPN

Navigate to your sandbox [web terminal](/docs/en/web-terminal) or [access it via SSH](/docs/en/ssh-access) and install OpenVPN and tmux (or similar):

```bash
sudo apt update && sudo apt install -y openvpn tmux
```

## 3. OpenVPN client configuration

Create the OpenVPN client configuration file for your Daytona sandbox. The configuration file should contain your client settings, certificates, and keys provided by your OpenVPN server administrator.

```bash
sudo nano client.ovpn
```

Define your OpenVPN client configuration into the file. The configuration should look similar to:

```bash
client
proto udp
explicit-exit-notify
remote <YOUR_OPENVPN_SERVER_IP> <YOUR_OPENVPN_SERVER_PORT>
dev tun
resolv-retry infinite
nobind
persist-key
persist-tun
remote-cert-tls server
verify-x509-name <YOUR_OPENVPN_SERVER_NAME> name
auth SHA256
auth-nocache
cipher AES-128-GCM
ignore-unknown-option data-ciphers
data-ciphers AES-128-GCM
ncp-ciphers AES-128-GCM
tls-client
tls-version-min 1.2
tls-cipher TLS-ECDHE-ECDSA-WITH-AES-128-GCM-SHA256
tls-ciphersuites TLS_AES_256_GCM_SHA384:TLS_AES_128_GCM_SHA256:TLS_CHACHA20_POLY1305_SHA256
ignore-unknown-option block-outside-dns
setenv opt block-outside-dns # Prevent Windows 10 DNS leak
verb 3
<ca>
-----BEGIN CERTIFICATE-----
<YOUR_OPENVPN_SERVER_CERTIFICATE>
-----END CERTIFICATE-----
</ca>
<cert>
-----BEGIN CERTIFICATE-----
<YOUR_OPENVPN_CLIENT_CERTIFICATE>
-----END CERTIFICATE-----
</cert>
<key>
-----BEGIN PRIVATE KEY-----
<YOUR_OPENVPN_CLIENT_PRIVATE_KEY>
-----END PRIVATE KEY-----
</key>
<tls-crypt-v2>
-----BEGIN OpenVPN tls-crypt-v2 client key-----
<YOUR_OPENVPN_TLS_CRYPT_V2_CLIENT_KEY>
-----END OpenVPN tls-crypt-v2 client key-----
</tls-crypt-v2>
```

Save the file by pressing `Ctrl+O`, then `Enter`, and exit by pressing `Ctrl+X`.

## 4. Start OpenVPN tunnel

Daytona requires the OpenVPN tunnel to be running in the background to connect the sandbox to the OpenVPN network. The recommended approach is to run it in a detached tmux (or similar) session:

```bash
tmux new -d -s openvpn 'sudo openvpn client.ovpn'
```

This starts the OpenVPN tunnel in a background tmux session that persists even if you disconnect from the sandbox.

## 5. Verify OpenVPN connection

To verify the OpenVPN connection, use the following command:

```bash
curl ifconfig.me
```

This will return the IP address of the sandbox connected to the OpenVPN network.

You've now successfully connected your Daytona sandbox to your OpenVPN network.
