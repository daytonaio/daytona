## Image

Represents an image definition for a Daytona sandbox.
Do not construct this class directly. Instead use one of its static factory methods,
such as `Image.base()`, `Image.debian_slim()`, or `Image.from_dockerfile()`.

**Attributes**:

- `dockerfile` _String_ - The generated Dockerfile for the image
- `context_list` _Array\<Context\>_ - List of context files for the image

### Class Methods

#### Image.base()

```ruby
def self.base(image)
```

Creates an Image from an existing base image.

**Parameters**:

- `image` _String_ - The base image to use

**Returns**:

- `Daytona::Image` - The image with the base image added

**Example:**

```ruby
image = Daytona::Image.base('python:3.12-slim-bookworm')
```

***

#### Image.debian_slim()

```ruby
def self.debian_slim(python_version = nil)
```

Creates a Debian slim image based on the official Python Docker image.

**Parameters**:

- `python_version` _String, nil_ - The Python version to use (e.g., '3.12'). Defaults to latest supported version.

**Returns**:

- `Daytona::Image` - The image with the Debian slim image added

**Example:**

```ruby
image = Daytona::Image.debian_slim('3.12')
```

***

#### Image.from_dockerfile()

```ruby
def self.from_dockerfile(path)
```

Creates an Image from an existing Dockerfile.

**Parameters**:

- `path` _String_ - The path to the Dockerfile

**Returns**:

- `Daytona::Image` - The image with the Dockerfile added

**Example:**

```ruby
image = Daytona::Image.from_dockerfile('Dockerfile')
```

### Instance Methods

#### pip_install()

```ruby
def pip_install(*packages, find_links: nil, index_url: nil, extra_index_urls: nil, pre: false, extra_options: '')
```

Adds commands to install packages using pip.

**Parameters**:

- `packages` _Array\<String\>_ - The packages to install
- `find_links` _Array\<String\>, nil_ - The find-links to use
- `index_url` _String, nil_ - The index URL to use
- `extra_index_urls` _Array\<String\>, nil_ - The extra index URLs to use
- `pre` _Boolean_ - Whether to install pre-release packages. Defaults to false.
- `extra_options` _String_ - Additional options to pass to pip

**Returns**:

- `Daytona::Image` - The image with the pip install commands added

**Example:**

```ruby
image = Daytona::Image.debian_slim('3.12').pip_install('requests', 'pandas')
```

## Building Complex Images

The Image class supports method chaining for building complex images:

```ruby
image = Daytona::Image
  .debian_slim('3.12')
  .pip_install('numpy', 'pandas', 'matplotlib')
  .add_local_file('config.json', '/home/daytona/config.json')
  .env({ 'MY_VAR' => 'value' })
  .workdir('/home/daytona')
  .run_commands('apt-get update', 'apt-get install -y vim')

# Use the image to create a sandbox
params = Daytona::CreateSandboxFromImageParams.new(
  image: image,
  language: Daytona::CodeLanguage::PYTHON
)
sandbox = daytona.create(params, on_snapshot_create_logs: ->(chunk) { print chunk })
```

## Supported Python Versions

The `debian_slim` method supports the following Python versions:

- 3.9
- 3.10
- 3.11
- 3.12
- 3.13

If no version is specified, the latest supported version is used.
