title: API Keys
---

DaytonaのAPIキーは、[Daytona API](/docs/ja/tools/api)へのリクエストを認証するために使用します。

[Configuration](/docs/ja/configuration)ページでは、`DAYTONA_API_KEY` 環境変数の設定方法を説明しています。

APIキーを管理するには、Dashboardの[Keys](https://app.daytona.io/dashboard/keys)セクションに移動します。`Create Key` をクリックして新しいキーを作成し、クリップボードにコピーします。

## 権限

新しい APIキー を追加する際、キーに付与する権限セットを選択するよう求められます:

| Permission | Description |
|------------|-------------|
| **`Sandboxes`** | サンドボックス（Daytonaが管理する隔離された一時的な実行環境）を作成・管理する権限を付与 |
| **`Snapshots`** | スナップショット（サンドボックス作成に使う事前設定済みの再利用可能なイメージ／テンプレート）を作成・管理する権限を付与 |
| **`Registries`** | レジストリを作成・管理する権限を付与 |
| **`Volumes`** | ボリューム（S3互換オブジェクトストレージをバックエンドとするFUSE共有ストレージマウント）を作成・管理する権限を付与 |

各権限には、より細かなスコープが含まれます:

- `write:*` は作成および更新
- `delete:*` はリソースの削除

例: `write:sandboxes`, `delete:sandboxes`
