Claude ガイドへようこそ。ここでは、さまざまなユースケースや目的に合わせて設計された、Claude と Daytona の多様な連携方法を紹介するガイドを見つけることができます。以下のガイドから、ニーズに合ったソリューションを見つけてください。

## ガイド \{#guides\}

#### <a href="/docs/ja/claude-agent-sdk-interactive-terminal-sandbox"><h4>リモートサンドボックスで動作する Claude Agent SDK にターミナルを接続する 🔗</h4></a> \{#connect-your-terminal-to-a-claude-agent-sdk-running-in-a-remote-sandbox\}

安全な Daytona のサンドボックス内で動作する Claude Code と、ターミナルから直接対話する方法を解説します。このガイドでは、Claude Agent SDK を用いて自律的に動作するコーディングエージェントを構築し、隔離された環境内でコーディングタスクをリアルタイムにやり取りしつつ実行できるようにする手順を示します。

#### <a href="/docs/ja/claude-agent-sdk-connect-service-sandbox"><h4>リモートサンドボックスで稼働する Claude Agent SDK にサービスを接続する 🔗</h4></a> \{#connect-your-service-to-a-claude-agent-sdk-running-in-a-remote-sandbox\}

このガイドでは、アプリケーション、サービス、またはカスタムエージェントを、リモートの Daytona サンドボックス内で安全に稼働している Claude エージェントに接続する方法を説明します。ここでは、Project Manager Agent（ローカル）と Developer Agent（サンドボックス内）という 2 つのエージェントで構成される自律型コーディングシステムの構築例を通じて、リモートサンドボックス環境内で稼働する Claude エージェントを必要とするあらゆる種類のサービスに対し、簡単に接続できるよう構成する方法を紹介します。

#### <a href="/docs/ja/claude-code-run-tasks-stream-logs-sandbox"><h4>タスクベース実行: Daytona サンドボックスで Claude Code タスクを実行し、ログをプログラム経由でストリーミングする 🔗</h4></a> \{#task-based-execution-run-claude-code-tasks-in-a-daytona-sandbox-and-stream-logs-programmatically\}

このガイドでは、Daytona サンドボックス内で Claude Code タスクを実行し、ログをプログラム経由でストリーミングするためにそのまま利用できるコードスニペットを提供します。コピー＆ペーストするだけで、Claude Code のワークフローを安全かつ効率的に自動化およびオーケストレーションできるようになります。
