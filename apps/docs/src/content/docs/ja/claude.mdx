Claude ガイドへようこそ。ここでは、さまざまなユースケースや目的に合わせて設計された、Claude と Daytona の多様な連携方法を紹介するガイドを見つけることができます。以下のガイドから、ニーズに合ったソリューションを見つけてください。

## ガイド \{#guides\}

#### <a href="/docs/ja/connect-terminal-agent-sandbox"><h4>リモートサンドボックスで動作する Claude Agent SDK にターミナルを接続する 🔗</h4></a> \{#connect-your-terminal-to-a-claude-agent-sdk-running-in-a-remote-sandbox\}

Daytona のサンドボックス環境内で、自律的に動作するコーディングエージェントを構築する方法を解説します。このエージェントは [Claude Agent SDK](https://platform.claude.com/docs/en/agent-sdk/overview) を使用して、ユーザーのプロンプトに従って動作します。

#### <a href="/docs/ja/claude-agent-sdk-connect-service-sandbox"><h4>リモートサンドボックスで稼働する Claude Agent SDK にサービスを接続する 🔗</h4></a> \{#connect-your-service-to-a-claude-agent-sdk-running-in-a-remote-sandbox\}

このガイドでは、アプリケーション、サービス、またはカスタムエージェントを、リモートの Daytona サンドボックス内で安全に稼働している Claude エージェントに接続する方法を説明します。ここでは、Project Manager Agent（ローカル）と Developer Agent（サンドボックス内）という 2 つのエージェントから成る自律型コーディングシステムを構築し、リモートサンドボックス環境内で稼働する Claude エージェントを必要とするあらゆる種類のサービスに、簡単に接続できるよう構成する方法を紹介します。
