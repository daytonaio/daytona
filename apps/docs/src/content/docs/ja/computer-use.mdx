コンピューター使用は、Daytona のサンドボックス内にあるデスクトップ環境のプログラムによる制御を可能にします。GUI の操作自動化やデスクトップアプリケーションのテストのために、マウス、キーボード、スクリーンショット、およびディスプレイ操作を提供します。

**Linux**、**Windows**、または **macOS** アプリケーションと対話する必要がある場合は、このインスタンスタイプの使用を推奨します。

:::caution[アルファ版へのアクセスが必要]
macOS および Windows 向けのコンピューター使用は現在 **非公開アルファ** 段階にあります。アクセスをリクエストするには、[Computer Use Windows Access Request Form](https://docs.google.com/forms/d/e/1FAIpQLSfoK-77-VpfsMubw8F4f1opCxIL1AyJUgnM0ONYup5hZ0RTvQ/viewform?usp=dialog) または [Computer Use macOS Access Request Form](https://docs.google.com/forms/d/e/1FAIpQLSc9xlGZ49OjWNkyzDPC9Ip3InMRR0ZXY3tcoD-PFQj3ck6gzQ/viewform?usp=sharing\&ouid=103304973264148733944) にご記入ください。送信後、当社チームがリクエスト内容を確認し、セットアップ手順および利用可能状況の詳細についてご連絡します。
:::

## よくあるユースケース \{#common-use-cases\}

* **GUI アプリケーションテスト** - ネイティブ macOS アプリケーションの操作を自動化し、ボタンのクリックやフォーム入力を行い、UI の動作を検証する
* **ビジュアルテストとスクリーンショット** - アプリケーションのスクリーンショットをキャプチャし、UI 状態を比較してビジュアル回帰テストを行う
* **デスクトップ自動化** - 繰り返し発生する macOS デスクトップ作業や GUI を介したファイル管理、複雑なワークフローを自動化する

## コンピューター使用を開始する \{#start-computer-use\}

サンドボックス内でコンピューター使用に関連するすべてのプロセス（Xvfb、xfce4、x11vnc、novnc）を起動します。

<Tabs syncKey="language">
  <TabItem label="Python" icon="seti:python">
    ```python
    result = sandbox.computer_use.start()
    print("Computer use processes started:", result.message)
    ```
  </TabItem>

  <TabItem label="TypeScript" icon="seti:typescript">
    ```typescript
    const result = await sandbox.computerUse.start();
    console.log('Computer use processes started:', result.message);
    ```
  </TabItem>

  <TabItem label="Ruby" icon="seti:ruby">
    ```ruby
    result = sandbox.computer_use.start
    puts "Computer use processes started: #{result.message}"
    ```
  </TabItem>

  <TabItem label="Go" icon="seti:go">
    ```go
    err := sandbox.ComputerUse.Start(ctx)
    if err != nil {
    	log.Fatal(err)
    }
    defer sandbox.ComputerUse.Stop(ctx)

    fmt.Println("Computer use processes started")
    ```
  </TabItem>

  <TabItem label="API" icon="seti:json">
    ```bash
    curl 'https://proxy.app.daytona.io/toolbox/{sandboxId}/computeruse/start' \
      --request POST
    ```
  </TabItem>
</Tabs>

### クリック \{#click\}

<Tabs syncKey="language">
  <TabItem label="Python" icon="seti:python">
    ```python

    result = sandbox.computer_use.mouse.click(100, 200)

    # Double click

    # Right click
    right_click = sandbox.computer_use.mouse.click(100, 200, "right")
    ```
  </TabItem>

  <TabItem label="TypeScript" icon="seti:typescript">
    ```typescript
    // Single left click
    const result = await sandbox.computerUse.mouse.click(100, 200);

    // Double click
    const doubleClick = await sandbox.computerUse.mouse.click(100, 200, 'left', true);

    // Right click
    const rightClick = await sandbox.computerUse.mouse.click(100, 200, 'right');
    ```
  </TabItem>

  <TabItem label="Ruby" icon="seti:ruby">
    ```ruby

    result = sandbox.computer_use.mouse.click(x: 100, y: 200)

    # Double click

    # Right click
    right_click = sandbox.computer_use.mouse.click(x: 100, y: 200, button: 'right')
    ```
  </TabItem>

  <TabItem label="Go" icon="seti:go">

:::tip[ストリーミングダウンロード]
すべての SDK の実装では、録画ファイルをメモリに全体を読み込まずに、ディスクに直接ストリーミングします。これにより、数百 MB 〜 数 GB クラスの大きな録画ファイルでも、メモリ不足になることなく効率的にダウンロードできます。

* **Python**: `httpx` を使用し、64KB チャンクでストリーミング
* **TypeScript**: バックプレッシャー制御付きの Node.js `pipeline()` を使用
* **Ruby**: `on_body` コールバック付きの Typhoeus ストリーミングを使用
* **Go**: 32KB の内部バッファを持つ `io.Copy()` を使用
  :::

### レコーディングダッシュボード \{#recording-dashboard\}

すべてのサンドボックスには、Web インターフェース経由で画面録画を管理できる組み込みのレコーディングダッシュボードが含まれています。ダッシュボードを使用すると、コードを書くことなく録画を閲覧、ダウンロード、削除できます。

レコーディングダッシュボードにアクセスするには:

1. Daytona Dashboard で自分のサンドボックスに移動します
2. 対象のサンドボックスのアクションメニュー（3 つの点）をクリックします
3. ドロップダウンメニューから **Screen Recordings** を選択します

レコーディングダッシュボードでは、次のことができます:

* すべての録画とそのメタデータ（名前、再生時間、ファイルサイズ、作成時刻）の一覧表示
* 録画内容を確認するための再生コントロール
* 録画をローカル環境に保存するためのダウンロード機能
* ストレージ使用量を管理するための削除オプション

:::tip
レコーディングダッシュボードはプライベートポートで動作し、自動的に保護されています。Daytona Dashboard からアクセスした場合、追加の認証は不要です。
:::
