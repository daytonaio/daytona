Daytonaは、サンドボックス内の`git`モジュールを通じて、Gitの標準サポートを提供します。

## 基本操作 \{#basic-operations\}

Daytona では、サンドボックス内の Git リポジトリのクローン、ステータス確認、および管理を行うためのメソッドが提供されています。

[ファイルシステム操作](/docs/ja/file-system-operations)と同様に、クローンの開始ディレクトリは現在のサンドボックスの作業ディレクトリです。Dockerfile に WORKDIR が指定されている場合はそれを使用し、指定されていない場合はユーザーのホームディレクトリが使用されます。たとえば `workspace/repo` は `/my-work-dir/workspace/repo` を意味します。なお、パスを `/` で始めれば、絶対パスの `workDir` を指定することもできます。

### リポジトリのクローン \{#clone-repositories\}

Daytona は、Git リポジトリをサンドボックスへクローンするためのメソッドを提供します。公開・非公開リポジトリ、特定ブランチのクローンに対応し、Personal Access Token による認証も可能です。

<Tabs syncKey="language">
  <TabItem label="Python" icon="seti:python">
    ```python
    # 基本的なクローン
    sandbox.git.clone(
        url="https://github.com/user/repo.git",
        path="workspace/repo"
    )

    # 認証付きクローン
    sandbox.git.clone(
        url="https://github.com/user/repo.git",
        path="workspace/repo",
        username="git",
        password="personal_access_token"
    )

    # 特定ブランチのクローン
    sandbox.git.clone(
        url="https://github.com/user/repo.git",
        path="workspace/repo",
        branch="develop"
    )
    ```
  </TabItem>

  <TabItem label="TypeScript" icon="seti:typescript">
    ```typescript

    await sandbox.git.clone(
        "https://github.com/user/repo.git",
        "workspace/repo"
    );

    // 認証付きクローン
    await sandbox.git.clone(
        "https://github.com/user/repo.git",
        "workspace/repo",
        undefined,
        undefined,
        "git",
        "personal_access_token"
    );

    // 特定ブランチのクローン
    await sandbox.git.clone(
        "https://github.com/user/repo.git",
        "workspace/repo",
        "develop"
    );
    ```
  </TabItem>

  <TabItem label="Ruby" icon="seti:ruby">
    ```ruby

      url: 'https://github.com/user/repo.git',
      path: 'workspace/repo'
    )

    sandbox.git.clone(
      url: 'https://github.com/user/repo.git',
      path: 'workspace/repo',
      username: 'git',
      password: 'personal_access_token'
    )

    # 特定ブランチのクローン
    sandbox.git.clone(
      url: 'https://github.com/user/repo.git',

    ```
  </TabItem>

  <TabItem label="Go" icon="seti:go">

    ```go
    if err != nil {
    	log.Fatal(err)
    }

    	options.WithUsername("git"),
    	options.WithPassword("personal_access_token"),
    )
    if err != nil {
    	log.Fatal(err)
    }

    	options.WithBranch("develop"),
    )
    if err != nil {
    	log.Fatal(err)
    }
    ```
  </TabItem>

### リポジトリの状態を取得 \{#get-repository-status\}

Daytona では、サンドボックス内の Git リポジトリの状態を確認するためのメソッドを提供しています。現在のブランチ、変更されたファイル、メインブランチに対して先行/遅延しているコミット数を取得できます。

<Tabs syncKey="language">
  <TabItem label="Python" icon="seti:python">
    ```python
    # リポジトリの状態を取得
    status = sandbox.git.status("workspace/repo")
    print(f"Current branch: {status.current_branch}")
    print(f"Commits ahead: {status.ahead}")
    print(f"Commits behind: {status.behind}")
    for file in status.file_status:
        print(f"File: {file.name}")

    # ブランチの一覧
    response = sandbox.git.branches("workspace/repo")
    for branch in response.branches:
        print(f"Branch: {branch}")
    ```
  </TabItem>

  <TabItem label="TypeScript" icon="seti:typescript">
    ```typescript
    // リポジトリの状態を取得
    const status = await sandbox.git.status("workspace/repo");
    console.log(`Current branch: ${status.currentBranch}`);
    console.log(`Commits ahead: ${status.ahead}`);
    console.log(`Commits behind: ${status.behind}`);
    status.fileStatus.forEach(file => {
        console.log(`File: ${file.name}`);
    });

    // ブランチの一覧
    const response = await sandbox.git.branches("workspace/repo");
    response.branches.forEach(branch => {
        console.log(`Branch: ${branch}`);
    });
    ```
  </TabItem>

  <TabItem label="Ruby" icon="seti:ruby">
    ```ruby
    # リポジトリの状態を取得
    status = sandbox.git.status('workspace/repo')
    puts "Current branch: #{status.current_branch}"
    puts "Commits ahead: #{status.ahead}"
    puts "Commits behind: #{status.behind}"
    status.file_status.each do |file|
      puts "File: #{file.name}"
    end

    # ブランチの一覧
    response = sandbox.git.branches('workspace/repo')
    response.branches.each do |branch|
      puts "Branch: #{branch}"
    end
    ```
  </TabItem>

  <TabItem label="Go" icon="seti:go">
    ```go
    // リポジトリの状態を取得
    status, err := sandbox.Git.Status(ctx, "workspace/repo")
    if err != nil {
    	log.Fatal(err)
    }
    fmt.Printf("Current branch: %s\n", status.CurrentBranch)
    fmt.Printf("Commits ahead: %d\n", status.Ahead)
    fmt.Printf("Commits behind: %d\n", status.Behind)
    for _, file := range status.FileStatus {
    	fmt.Printf("File: %s\n", file.Path)
    }

    // ブランチの一覧
    branches, err := sandbox.Git.Branches(ctx, "workspace/repo")
    if err != nil {
    	log.Fatal(err)
    }
    for _, branch := range branches {
    	fmt.Printf("Branch: %s\n", branch)
    }
    ```
  </TabItem>

  <TabItem label="API" icon="seti:json">
    ```bash
    curl 'https://proxy.app.daytona.io/toolbox/{sandboxId}/git/status?path='
    ```
  </TabItem>
</Tabs>

## 変更をステージングする \{#stage-changes\}

Daytona は、Git リポジトリ内の変更をステージするためのメソッドを提供します。特定のファイルのみ、またはすべての変更をステージし、メッセージ付きでコミットできます。次のコードスニペットでは、`file.txt` ファイルと `src` ディレクトリをステージします。

<Tabs syncKey="language">
  <TabItem label="Python" icon="seti:python">
    ```python
    # Stage a single file
    sandbox.git.add("workspace/repo", ["file.txt"])

    # Stage multiple files
    sandbox.git.add("workspace/repo", [
        "src/main.py",
        "tests/test_main.py",
        "README.md"
    ])
    ```
  </TabItem>

  <TabItem label="TypeScript" icon="seti:typescript">
    ```typescript
    // Stage a single file
    await git.add('workspace/repo', ['file.txt']);
    // Stage whole repository
    await git.add('workspace/repo', ['.']);
    ```
  </TabItem>

  <TabItem label="Ruby" icon="seti:ruby">
    ```ruby
    # Stage a single file
    sandbox.git.add('workspace/repo', ['file.txt'])
    ```
  </TabItem>

  <TabItem label="Go" icon="seti:go">
    ```go
    // Stage a single file
    err := sandbox.Git.Add(ctx, "workspace/repo", []string{"file.txt"})
    if err != nil {
    	log.Fatal(err)
    }

    // Stage multiple files
    err = sandbox.Git.Add(ctx, "workspace/repo", []string{
    	"src/main.py",
    	"tests/test_main.py",
    	"README.md",
    })
    if err != nil {
    	log.Fatal(err)
    }

    // Stage whole repository
    err = sandbox.Git.Add(ctx, "workspace/repo", []string{"."})
    if err != nil {
    	log.Fatal(err)
    }
    ```
  </TabItem>

  <TabItem label="API" icon="seti:json">
    ```bash
    curl 'https://proxy.app.daytona.io/toolbox/{sandboxId}/git/add' \
      --request POST \
      --header 'Content-Type: application/json' \
      --data '{
      "files": [
        ""
      ],
      "path": ""
    }'
    ```
  </TabItem>
</Tabs>

## 変更をコミットする \{#commit-changes\}

Daytona は、Git リポジトリ内の変更をコミットするためのメソッドを提供します。コミットメッセージ、作成者名、メールアドレスを指定してコミットできます。次のコードスニペットでは、メッセージ `Update documentation`、作成者 `John Doe`、メールアドレス `john@example.com` で変更をコミットします。

<Tabs syncKey="language">
  <TabItem label="Python" icon="seti:python">
    ```python
    # 変更をステージしてコミットする
    sandbox.git.add("workspace/repo", ["README.md"])
    sandbox.git.commit(
        path="workspace/repo",
        message="Update documentation",
        author="John Doe",
        email="john@example.com",
        allow_empty=True
    )
    ```
  </TabItem>

  <TabItem label="TypeScript" icon="seti:typescript">
    ```typescript
    // 変更をステージしてコミットする
    await git.add('workspace/repo', ['README.md']);
    await git.commit(
      'workspace/repo',
      'Update documentation',
      'John Doe',
      'john@example.com',
      true
    );
    ```
  </TabItem>

  <TabItem label="Ruby" icon="seti:ruby">
    ```ruby
    # 変更をステージしてコミットする
    sandbox.git.add('workspace/repo', ['README.md'])
    sandbox.git.commit('workspace/repo', 'Update documentation', 'John Doe', 'john@example.com', true)
    ```
  </TabItem>

  <TabItem label="Go" icon="seti:go">

## リモート操作 \{#remote-operations\}

Daytona は、Git のリモートリポジトリを扱うためのメソッドを提供します。リモートリポジトリとの間で変更を push および pull できます。

### 変更をプッシュ \{#push-changes\}

Daytona は、変更をリモートリポジトリにプッシュするためのメソッドを提供します。次のスニペットは、変更をパブリックリポジトリにプッシュする例です。

<Tabs syncKey="language">
  <TabItem label="Python" icon="seti:python">
    ```python
    # 認証なしでプッシュ（パブリックリポジトリまたは SSH 用）
    sandbox.git.push("workspace/repo")

    # 認証ありでプッシュ
    sandbox.git.push(
        path="workspace/repo",
        username="user",
        password="github_token"
    )
    ```
