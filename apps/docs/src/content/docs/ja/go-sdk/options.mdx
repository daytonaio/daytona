# オプション \{#options\}

```go
import "github.com/daytonaio/daytona/libs/sdk-go/pkg/options"
```

options パッケージは、SDK の各種操作を設定するための関数型オプション型を提供します。

このパッケージは functional options パターンを用いて、オプションパラメータを設定するためのクリーンで拡張性の高い API を提供します。各オプション関数は、対応する options 構造体を変更するクロージャを返します。

### 使用方法 \{#usage\}

Options は可変長引数として SDK メソッドに渡します。

```
err := sandbox.Git.Clone(ctx, url, path,
    options.WithBranch("develop"),
    options.WithUsername("user"),
    options.WithPassword("token"),
)
```

### 汎用的な Apply 関数 \{#generic-apply-function\}

[Apply](#Apply) 関数は新しいオプション構造体を作成し、指定されたすべてのオプション関数をその構造体に適用します。

```
opts := options.Apply(
    options.WithBranch("main"),
    options.WithUsername("user"),
)
// opts.Branch == "main", opts.Username == "user"
```

## func Apply \{#func-apply\}

```go
func Apply[T any](opts ...func(*T)) *T
```

Apply は、型 T の新しいインスタンスを作成し、指定されたすべてのオプション関数を適用します。

このジェネリック関数により、さまざまなオプション型に対して関数ベースのオプションを適用するための一貫したパターンが提供されます。まず T のゼロ値インスタンスを割り当て、その後、各オプション関数を順番に適用します。

例:

```
opts := options.Apply(
    options.WithBranch("main"),
    options.WithUsername("user"),
)
```

<a name="WithAllowEmpty" />

## func WithAllowEmpty \{#func-withallowempty\}

```go
func WithAllowEmpty(allowEmpty bool) func(*GitCommit)
```

WithAllowEmpty を指定すると、ステージされた変更がない場合でもコミットを作成できます。

これは、実際のコード変更がなくても CI/CD パイプラインをトリガーしたり、履歴上の特定の時点をマークしたりするのに有用です。

例:

```
resp, err := sandbox.Git.Commit(ctx, path, "Trigger rebuild", author, email,
    options.WithAllowEmpty(true),
)
```

func WithLogChannel(logChannel chan string) func(*CreateSandbox)
```

WithLogChannel は、サンドボックス作成時のビルドログを受信するためのチャネルを提供します。

ビルドが必要なカスタムイメージからサンドボックスを作成する場合、ビルドログは指定されたチャネルにストリーミングされます。ストリーミングが完了すると、そのチャネルはクローズされます。ビルドが不要な場合はログは送信されず、そのチャネルは使用されません。

    options.WithLogChannel(logChan),
)
```

<a name="WithWaitForStart" />

## func WithWaitForStart \{#func-withwaitforstart\}

```go
func WithWaitForStart(waitForStart bool) func(*CreateSandbox)
```

WithWaitForStart は、[daytona.Client.Create] がサンドボックスが started 状態に到達するまで待機してから戻るかどうかを制御します。

true（デフォルト）の場合、Create はサンドボックスが完全に起動して使用可能になるまで処理をブロックします。false の場合、サンドボックスが作成されるとすぐに Create は戻りますが、その時点では pending または building 状態の場合があります。

例:

```
// サンドボックスの起動を待たずにすぐに戻る
sandbox, err := client.Create(ctx, params,
    options.WithWaitForStart(false),
)
```
