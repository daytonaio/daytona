このガイドでは、安全な Daytona サンドボックス内で [OpenCode](https://opencode.ai/) 自律型コーディングエージェントを実行する手順を説明します。OpenCode サーバーはサンドボックス内で完全に分離された状態で動作し、スクリプトは [OpenCode SDK](https://www.npmjs.com/package/@opencode-ai/sdk) を使ってサーバーに接続します。

このエージェントは、フルスタック Web アプリを開発し、任意の言語でコードを書き、依存関係をインストールし、スクリプトを実行できます。また、開発サーバーを起動・管理し、稼働中のアプリ向けに[プレビューリンク](/docs/ja/preview/)を生成することもできます。

### 2. プロジェクトのセットアップ \{#2-project-setup\}

#### リポジトリをクローンする \{#clone-the-repository\}

まず、Daytonaの[リポジトリ](https://github.com/daytonaio/daytona.git)をクローンし、OpenCode SDK のサンプルディレクトリに移動します。

```bash
git clone https://github.com/daytonaio/daytona.git
cd daytona/guides/typescript/opencode/opencode-sdk
```

#### 環境を設定する \{#configure-environment\}

[Daytona Dashboard](https://app.daytona.io/dashboard/keys) から API キーを取得します。

`.env.example` を `.env` にコピーし、取得したキーを追加します。

```bash
DAYTONA_API_KEY=your_daytona_key
```

#### ローカル環境での利用 \{#local-usage\}

:::note[Node.js のバージョン]
この例を実行するには Node.js 18 以上が必要です。続行する前に、お使いの環境がこの要件を満たしていることを確認してください。
:::

依存関係をインストールして、このサンプルを実行します。

```bash
npm install
npm run start
```

エージェントが起動し、プロンプトの入力待ち状態になります。

### 3. スクリプトを理解する \{#3-understanding-the-script\}

OpenCode はクライアント・サーバーモデルを採用しています。[サーバー](https://opencode.ai/docs/server/) はコーディングエージェント、セッション、および設定を管理します。クライアントは [HTTP 経由](https://opencode.ai/docs/server/#apis) でサーバーと通信し、セッション内でプロンプトを実行してストリーミング形式のレスポンスを受信します。

#### 初期化 \{#initialization\}

起動時、このスクリプトは次の処理を行います。

1. 新しい [Daytona サンドボックス](/docs/ja/sandboxes) を作成します。
2. サンドボックス内に OpenCode をグローバルにインストールします。
3. サンドボックス内で OpenCode サーバーを起動します。
4. サーバーのプレビュー URL を取得し、そのベース URL を使ってクライアントを作成します。
5. `readline` ループに入り、プロンプトをサーバーに送信してストリーミング形式のレスポンスを受信します。
6. Ctrl+C が押されたときにサンドボックスを削除し、終了します。

#### OpenCode Server \{#opencode-server\}

OpenCode サーバーはサンドボックス内で実行され、すべてのエージェントの処理（LLM 呼び出し、ツール（bash、write など）、コード実行）を扱います。スクリプトでは、次のようにしてこのサーバーを起動します：

```ts
const envVar = injectEnvVar('OPENCODE_CONFIG_CONTENT', opencodeConfig)
const command = await sandbox.process.executeSessionCommand(sessionId, {
  command: `${envVar} opencode serve --port ${PORT} --hostname ${HOSTNAME}`,
  runAsync: true,
})
```

システムプロンプトを含むカスタム設定が注入されます。システムプロンプトは、エージェントがサンドボックス内で実行されていること、ファイル操作には `/home/daytona` を使用すること、および正しいプレビューリンクを提供できるようにプレビューURLのパターンをエージェントに指示します。

#### OpenCode クライアント \{#opencode-client\}

クライアントはローカルマシン上で動作し、[OpenCode SDK](https://opencode.ai/docs/sdk/) を使ってサーバーと通信します。`Session.create` は SDK クライアントを初期化し、イベントストリームを購読して、エージェントの動作中にクライアントがツールのアクティビティ（例: 「✓ Run: …」「📝 Add …」）を表示できるようにします。

```ts
const client = createOpencodeClient({ baseUrl })
const sessionRes = await client.session.create({ body: { title: 'Daytona query' } })
const sessionId = sessionRes.data?.id
if (!sessionId) throw new Error('Failed to create OpenCode session:' + sessionRes.error)
const events = await client.event.subscribe()
```

プロンプトを送信するには、クライアントが `session.prompt` を呼び出し、既存のストリームからのイベントを処理してツールの実行状況を表示します。

```ts
const promptPromise = this.client.session.prompt({
  path: { id: this.sessionId },
  body: { parts: [{ type: 'text', text: query } satisfies TextPartInput] },
})

for await (const event of takeUntil(this.events.stream, promptPromise)) {
  printEvent(this.sessionId, event)
}
```

#### メインループ \{#main-loop\}

サーバーの準備が整うと、スクリプトはセッションを作成し、readline ループを実行します。

```ts
const session = await Session.create(baseUrl)
const rl = readline.createInterface({ input: process.stdin, output: process.stdout })

while (true) {
  const query = await new Promise<string>((resolve) => rl.question('User: ', resolve))
  if (!query.trim()) continue
  await session.runQuery(query)
}
```

`readline` ループはユーザー入力を待ち、エージェントへ送り、その応答を表示します。

**主な利点:**

* Daytona サンドボックス内での、安全で分離された環境での実行
* ターミナルでの OpenCode SDK クライアント利用（サーバーはサンドボックス内で稼働）
* 75 以上の LLM プロバイダーをサポート
* すべてのエージェントコードの実行はサンドボックス内で行われる
* デプロイ済みサービス向けプレビューリンクの自動生成
* Daytona 固有のワークフロー向けにエージェントのカスタム構成が可能
