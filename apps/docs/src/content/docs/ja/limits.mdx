---
title: 制限
---

Daytonaは、すべての組織で公平な使用と安定性を確保するためにリソース制限を実施しています。あなたの組織は、以下で構成されるコンピュートプールにアクセスできます：

- **vCPU** — 利用可能な総CPU コア数
- **メモリ** — 利用可能な総RAM容量
- **ストレージ** — 利用可能な総ディスク容量

これらのリソースはプールされ、実行中のすべてのSandbox間で共有されます。  
同時に実行できるSandboxの数は、それぞれがどれだけのCPU、RAM、ストレージを使用するかによって決まります。リソース使用量の設定と見積もり方法については、[Sandbox Management docs](https://www.daytona.io/docs/sandbox-management/)をご覧ください。

## ティアと制限の増加

組織は認証ステータスに基づいて自動的にティアに配置されます。  
以下の手順を完了することで、より高い制限を解除できます：

| ティア   | リソース (vCPU / RAM / ストレージ) | アクセス要件                                                                                                     |
| ------ | -------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| ティア 1 | 10 / 10GiB / 30GiB               | メール認証済み                                                                                                          |
| ティア 2 | 100 / 200GiB / 300GiB            | クレジットカード連携済み、$25チャージ、[GitHub連携済み](https://www.daytona.io/docs/linked-accounts#how-to-link-an-account)。 |
| ティア 3 | 250 / 500GiB / 2000GiB           | ビジネスメール認証済み、電話認証済み、$500チャージ。                                                                   |
| ティア 4 | 500 / 1000GiB / 5000GiB          | 30日ごとに$2000チャージ。                                                                                             |
| カスタム | カスタム制限                    | [support@daytona.io](mailto:support@daytona.io)にお問い合わせください                                                                 |

より高いティアの条件を満たしたら、[ダッシュボード](https://app.daytona.io/dashboard/limits)の「Upgrade」ボタンをクリックしてアップグレードしてください。

:::tip

### 使用量を動的に管理

異なるサンドボックスの状態が利用可能なリソースに与える影響は以下の通りです：

- **実行中：** コンピュート、メモリ、ストレージを使用します。
- **停止中：** コンピュートとメモリを解放しますが、ストレージは使用します。
- **アーカイブ済み：** データをコールドストレージに移動してすべてのリソースを解放します。
- **削除済み：** 完全に削除され、すべてのリソースが解放されます。

:::

現在の使用量と制限は[ダッシュボード](https://app.daytona.io/dashboard/limits)で確認できます。

## さらに必要ですか？

より高い制限や特別な制限が必要な場合は、[support@daytona.io](mailto:support@daytona.io)までお問い合わせください。
