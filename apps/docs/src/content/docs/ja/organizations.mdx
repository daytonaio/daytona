Daytona では、リソースをまとめて協働を可能にするための単位として組織を提供しています。ユーザーは、個人用のパーソナル組織で単独で作業することも、コラボレーティブ組織で共同作業することもできます。

[Daytona Dashboard ↗](https://app.daytona.io/dashboard) にアクセスして、組織を管理します。

## 個人用組織 と 共同作業用組織 の違い \{#personal-vs-collaborative-organizations\}

すべての Daytona ユーザーは、単独利用や検証に最適な **個人用組織** から始まります。**共同作業用組織** はユーザーが手動で作成し、共有アクセスと制御により、会社全体でのコラボレーションを想定して設計されています。

| **機能**           | **個人用組織**                   | **共同作業用組織**                               |
| ------------------ | -------------------------------- | ---------------------------------------------- |
| **作成**           | サインアップ時に自動             | ユーザーが手動で作成                             |
| **メンバー**       | 単一ユーザーのみ                 | 複数ユーザー（招待制）                           |
| **アクセス制御**   | ロールや権限なし                 | リソース単位のきめ細かな割り当てを伴うロール     |
| **課金**           | 個々のユーザーに紐づく           | チームメンバー間で共有                           |
| **ユースケース**   | 個人テスト、小規模プロジェクト   | 会社／チームでの開発および本番運用               |
| **クォータ範囲**   | ユーザー単位                     | 全メンバーで共有                                 |
| **削除可否**       | いいえ                           | はい（オーナーが可能）                           |

ユーザーは [Daytona Dashboard ↗](https://app.daytona.io/dashboard) のサイドバーにあるドロップダウンから、個人用組織と共同作業用組織を切り替えられます。各組織には、それぞれ独自のサンドボックス、API キー、リソースクォータがあります。

## 組織ロール \{#organization-roles\}

組織内のユーザーは、次のいずれか 2 種類のロールを持つことができます。

1. **Owners** は、組織とそのリソースに対する完全な管理アクセス権を持ちます。組織の Owner は、管理操作を実行できます。
2. **Members** は、組織に対する管理アクセス権を持たず、組織リソースへのアクセスは [**Assignments**](#available-assignments) に基づいて決まります。

## 管理アクション \{#administrative-actions\}

組織のオーナーは、次のような管理アクションを実行できます：

* [新規ユーザーを組織に招待](#inviting-new-users)
* [保留中の招待を管理](#managing-invitations)
* 組織内ユーザーの[ロール](#organization-roles)を変更
* 組織メンバーの[アサインメント](#available-assignments)を更新
* 組織からユーザーを削除
* 監査ログを確認
* [組織を削除](#organization-settings)

## 利用可能なアサインメント \{#available-assignments\}

利用可能なアサインメントは次のとおりです。

| アサインメント          | 説明                                                              |
| ----------------------- | ----------------------------------------------------------------- |
| **`Viewer (required)`** | 組織内のすべてのリソースに対する読み取り権限（必須）              |
| **`Developer`**         | 組織内でサンドボックスおよびキーを作成できる権限                  |
| **`Sandboxes Admin`**   | 組織内のサンドボックスに対する管理者権限                          |
| **`Snapshots Admin`**   | 組織内のスナップショットに対する管理者権限                        |
| **`Registries Admin`**  | 組織内のレジストリに対する管理者権限                              |
| **`Volumes Admin`**     | 組織内のボリュームに対する管理者権限                              |
| **`Super Admin`**       | 組織内のすべてのリソースに対するフルアクセス権限                  |
| **`Auditor`**           | 組織内の監査ログへのアクセス権限                                  |
| **`Infrastructure Admin`** | 組織内のインフラストラクチャに対する管理者権限                |

## メンバー管理 \{#manage-members\}

### 新しいユーザーを招待する \{#invite-new-users\}

組織の **Owner** として、新しいユーザーを組織に招待するには:

1. [Members ↗](https://app.daytona.io/dashboard/members) に移動します
2. **Invite Member** ボタンをクリックします
3. 招待したいユーザーのメールアドレスを入力します
4. 新しいユーザーの[ロールを選択](#organization-roles)します。**`Member`** ロールを選択する場合は、そのユーザーの[アサインメント](#available-assignments)を設定します

### ユーザーを削除する \{#remove-users\}

組織の **Owner** は、次の手順で組織からユーザーを削除できます。

1. [Members ↗](https://app.daytona.io/dashboard/members) に移動します
2. 削除したいユーザーの横にある **Remove** ボタンをクリックします
3. **Remove** ボタンをクリックして削除を確定します

## 招待の管理 \{#manage-invitations\}

他の組織への保留中の招待を表示するには、サイドバー下部のドロップダウンを展開し、[Invitations ↗](https://app.daytona.io/dashboard/invitations) ページに移動します。

組織への招待を受諾すると、その組織に割り当てられたリソースのクォータにアクセスできるようになり、新しい [API key](/docs/ja/api-keys) を発行してサンドボックスを作成できます。

## 組織の設定 \{#organization-settings\}

Dashboard の [Settings](https://app.daytona.io/dashboard/settings) サブページでは、組織 ID と名前を確認でき、不要になった場合は組織を削除できます。この操作は取り消せないため、十分に注意して実行してください。個人用の組織は既定で作成されており、削除できません。
