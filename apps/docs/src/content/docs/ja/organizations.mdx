DaytonaのOrganizationsは、リソースをグループ化し、コラボレーションを可能にするために使用されます。ユーザーは個人のPersonal Organizationで個別に作業したり、Collaborative Organizationで共同作業を行うことができます。

### Personal vs Collaborative Organizations

すべてのDaytonaユーザーは**Personal Organization**から始まります。これは個人使用や実験に最適です。**Collaborative Organizations**は手動で作成され、共有アクセスと制御を備えた会社全体でのコラボレーション用に設計されています。

| 機能               | Personal Organization            | Collaborative Organization                     |
| ------------------ | -------------------------------- | ---------------------------------------------- |
| **作成**           | サインアップ時に自動             | ユーザーが手動で作成                           |
| **メンバー**       | 単一ユーザーのみ                 | 複数ユーザー（招待ベース）                     |
| **アクセス制御**   | ロールや権限なし                 | 詳細なリソースベースのAssignmentsを持つロール  |
| **請求**           | 個人ユーザーに紐付け             | チームメンバー間で共有                         |
| **使用例**         | 個人テスト、小規模プロジェクト   | 会社/チーム開発と本番環境                      |
| **クォータ範囲**   | ユーザーごと                     | 全メンバー間で共有                             |
| **削除可能**       | ❌ いいえ                        | ✅ はい（Ownerによる）                         |

ユーザーはサイドバーのドロップダウンを使用して、Personal OrganizationとCollaborative Organizationを切り替えることができます。各Organizationには独自のサンドボックス、APIキー、リソースクォータがあります。

## 組織の役割

組織内のユーザーは、`Owner`と`Member`という2つの異なる**役割**のいずれかを持つことができます。`Owner`は組織とそのリソースに対する完全な管理アクセス権を持ちます。`Member`は組織に対する管理アクセス権を持たず、組織リソースへのアクセスは**割り当て**に基づきます。

### 管理アクション

組織の`Owner`は以下のような管理アクションを実行できます：

- 組織に新しいユーザーを招待する
- 保留中の招待を管理する
- 組織内のユーザーの役割を変更する
- 組織メンバーの割り当てを更新する
- 組織からユーザーを削除する
- 監査ログを検査する
- 組織を削除する

## 新しいユーザーの招待

Organization `Owner`として、あなたのOrganizationに新しいユーザーを招待するには、_Membersページ_に移動し、
_Invite Member_をクリックして、招待したいユーザーのメールアドレスを入力し、**Role**を選択します。
`Member`ロールを選択した場合、その**Assignments**も定義できます。

## 利用可能な割り当て

利用可能な**割り当て**のリストには以下が含まれます：

| 割り当て                    | 説明                                                 |
| ----------------------- | ------------------------------------------------------------------- |
| **`Viewer (required)`** | 組織内のすべてのリソースへの読み取りアクセスを付与します                             |
| **`Developer`**         | 組織内でサンドボックスとキーを作成する機能を付与します |
| **`Sandboxes Admin`**   | 組織内のサンドボックスへの管理者アクセスを付与します                |
| **`Snapshots Admin`**   | 組織内のスナップショットへの管理者アクセスを付与します                |
| **`Registries Admin`**  | 組織内のレジストリへの管理者アクセスを付与します               |
| **`Volumes Admin`**     | 組織内のボリュームへの管理者アクセスを付与します                  |
| **`Super Admin`**       | 組織内のすべてのリソースへのフルアクセスを付与します             |
| **`Auditor`**           | 組織内の監査ログへのアクセスを付与します                     |

## 招待の管理

他の組織への参加の保留中の招待を表示するには、ユーザーはサイドバーの下部にあるドロップダウンを展開し、_招待_をクリックして_招待ページ_に移動できます。ユーザーが組織への参加招待を受け入れると、その組織に割り当てられたリソースクォータにアクセスできるようになり、新しいAPIキーを発行してサンドボックスを作成することで続行できます。

## 設定

ダッシュボードの設定サブページでは、組織IDと名前を確認し、不要になった場合は組織を削除することができます。この操作は元に戻すことができませんので、慎重に進めてください。個人組織はデフォルトで存在し、削除することはできません。
