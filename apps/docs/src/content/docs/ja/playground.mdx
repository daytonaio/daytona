---
title: プレイグラウンド
---

Daytona は、サンドボックスの作成、SDK 操作の実行、Daytona の機能の確認をブラウザーから直接行えるインタラクティブな環境（「Playground」）を提供します。

* **インタラクティブなサンドボックス作成**: カスタムリソースを指定してサンドボックスを構成・作成できます
* **コードスニペット生成**: [管理](#management) セクションで構成した値に基づいて、SDK コードサンプルを自動生成します
* [Web terminal](/docs/ja/web-terminal): 組み込みの Web ターミナルを通じて、サンドボックス内でコマンドを直接実行します
* [VNC access](/docs/ja/vnc-access): [コンピューター使用](/docs/ja/computer-use) 機能を使って GUI アプリケーションを操作できます

## ダッシュボードからのアクセス \{#access-from-dashboard\}

Playground にアクセスするには、[Daytona Dashboard ↗](https://app.daytona.io/dashboard/playground) に移動します。

Playground は [サンドボックス](#sandbox)、[Terminal](#terminal)、[VNC](#vnc) の 3 つのタブで構成されています。これら 3 つのタブはすべて、同じアクティブなサンドボックス上で動作します。

## サンドボックス \{#sandbox\}

サンドボックスタブは、サンドボックスを設定し、SDK 操作を実行するための対話型インターフェースを提供します。左側のパネルには、折りたたみ可能なセクションに整理された設定可能なパラメーターが表示されます。右側のパネルには、自動生成されたコードスニペットが表示され、確認・コピー・実行できます。

### 管理 \{#management\}

管理セクションには、サンドボックスを構成するために使用されるパラメーターが含まれます。これらのパラメーターを編集して、サンドボックスとそのリソースをカスタマイズできます。

* [sandbox language](/docs/ja/sandboxes#multiple-runtime-support): サンドボックスで使用するプログラミング言語ランタイムを選択します
* [sandbox resources](/docs/ja/sandboxes#resources): サンドボックスに割り当てるリソースを構成します
* [sandbox lifecycle](/docs/ja/sandboxes#sandbox-lifecycle): サンドボックスのライフサイクルポリシーを設定します

:::note
管理セクションでは、既存のプレイグラウンド用サンドボックスを置き換える形で、**Run** を明示的にクリックした場合にのみサンドボックスが作成されます。[Terminal](#terminal) または [VNC](#vnc) タブでは、アクティブなサンドボックスが存在しない場合、設定済みのパラメーターを使用してプレイグラウンドが自動的にサンドボックスを作成します。
:::

### ファイルシステム \{#file-system\}

ファイルシステムセクションでは、サンドボックス内のファイルやディレクトリを管理するための操作を提供します。次の操作を使用して、サンドボックス内のファイルやディレクトリを操作できます。

* **新しいディレクトリを作成**: ディレクトリのパスとそのパーミッションを設定します
* **ファイルとディレクトリを一覧表示**: ディレクトリのパスを設定します
* **ファイルを削除**: ファイルのパスと、ディレクトリを削除するチェックボックスを設定します

詳細については、[ファイルシステム操作](/docs/ja/file-system-operations)を参照してください。

### Git operations \{#git-operations\}

「Git operations」セクションでは、サンドボックス内の Git リポジトリを管理するための操作を提供します。次の操作を使用して、リポジトリのクローン作成、ステータス取得、およびブランチ一覧の取得を行えます。

* **Clone a Git repository**: リポジトリの URL、クローン先、ブランチ、コミット ID、認証情報を指定します
* **Get repository status**: リポジトリのステータスを取得します
* **List branches**: リポジトリ内のブランチ一覧を取得します

詳細については、[Git operations](/docs/ja/git-operations) を参照してください。

### プロセスとコード実行 \{#process-and-code-execution\}

この「プロセスとコード実行」セクションでは、サンドボックス内でコードスニペットやシェルコマンドを直接実行するための操作を提供します。シェルコマンドは固定ですが、コードスニペットは、選択した [sandbox language](/docs/ja/sandboxes#multiple-runtime-support) パラメータに応じて自動的に切り替わります。

詳細については、[プロセスとコード実行](/docs/ja/process-code-execution)を参照してください。

## ターミナル \{#terminal\}

ターミナルタブでは、サンドボックスに接続された Web ベースのターミナルが提供されます。これにより、コマンドの実行、ファイルの表示、デバッグのために、サンドボックス環境へコマンドラインから直接アクセスできます。

ターミナルはポート `22222` 上で動作し、サンドボックスが稼働している限りアクティブな状態を保ちます。サンドボックスが一定時間操作されずに停止した場合は、ターミナルへのアクセスを復元するためにサンドボックスを再起動してください。

:::note
ターミナルアクセスは、認証済みの [組織](/docs/ja/organizations) メンバーに限定されています。
:::

詳細については、[ウェブターミナル](/docs/ja/web-terminal) を参照してください。

## VNC \{#vnc\}

VNC タブを使用すると、サンドボックスのグラフィカルなデスクトップ環境にアクセスでき、GUI アプリケーションの操作や [コンピューター使用](/docs/ja/computer-use) 機能のテストを行えます。左側のペインにはセクションごとに整理された操作用コントロールがあり、右側のペインにはデスクトップビューが表示されます。

詳細については、[VNC access](/docs/ja/vnc-access) を参照してください。

### ディスプレイ \{#display\}

「Display」セクションでは、サンドボックスのデスクトップ環境を確認・操作するためのオプションを提供します。

* **Get display information**: 利用可能なディスプレイに関する情報を取得します
* **List open windows**: 現在開いているウィンドウの一覧を取得します

詳細については、[display operations](/docs/ja/computer-use#display-operations) を参照してください。

### キーボード \{#keyboard\}

「キーボード」セクションでは、サンドボックスにキーボード入力を送信するためのオプションを提供します。

* **ホットキーの組み合わせを送信**: サンドボックスにホットキーの組み合わせを送信します
* **キーを押す**: 任意の修飾キーを付けてキーを押します
* **テキストを入力**: アクティブなウィンドウにテキストを入力します

詳細については、[キーボード操作](/docs/ja/computer-use#keyboard-operations) を参照してください。

### マウス \{#mouse\}

このセクションでは、サンドボックス内でのマウス入力を制御するためのオプションが用意されています。

* **Click**: 指定した位置をクリックする
* **Drag**: ある位置から別の位置までドラッグする
* **Move**: カーソルを指定した位置に移動する
* **Get cursor position**: 現在のカーソル位置を取得する

詳細については、[mouse operations](/docs/ja/computer-use#mouse-operations) を参照してください。

### スクリーンショット \{#screenshot\}

このスクリーンショットセクションでは、サンドボックスのデスクトップのスクリーンショットを撮影するための各種オプションを提供します。

* **スクリーンショットを撮影**: 形式、スケール、品質、カーソルの表示有無を選択し、画面全体または特定の領域をキャプチャします

詳細については、[スクリーンショット操作](/docs/ja/computer-use#screenshot-operations)を参照してください。
