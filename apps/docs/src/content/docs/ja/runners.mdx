title: 顧客管理コンピュート
---

ランナーは Daytona のコンピュートプレーンを支えるマシンであり、サンドボックスのワークロードを実行するための基盤インフラストラクチャを提供します。各ランナーには次の責務があります：

* **Workload execution**: サンドボックスのワークロードを実行する
* **Resource management**: CPU、メモリ、ディスクリソースを割り当ておよび監視する
* **Health reporting**: Daytona コントロールプレーンに対してメトリクスとヘルスステータスを継続的に報告する
* **Network connectivity**: サンドボックス向けのネットワーク、プロキシ接続、SSH アクセスを管理する

[shared](/docs/ja/regions#shared-regions) と [dedicated](/docs/ja/regions#dedicated-regions) のリージョンにあるランナーは、プロビジョニングやメンテナンスから、監視やスケーリングに至るまで、Daytona によって完全に管理されます。カスタムリージョンでは、利用者自身がランナーマシンを用意し、その管理と運用に責任を持ちます。

:::caution
カスタムランナーは現在実験的な機能であり、今後のリリースで変更される可能性があります。
アクセスを希望される場合は、[support@daytona.io](mailto:support@daytona.io) までお問い合わせください。
:::

## カスタムリージョン \{#custom-regions\}

カスタムリージョンは組織によって作成および管理され、独自のランナーマシンを利用して、各リージョン内のコンピュートリソースを個別にスケールできます。これにより、データのローカリティ、コンプライアンス、およびインフラストラクチャ構成を最大限に制御できます。

さらに、カスタムリージョンにはリソースの同時利用に関する制限が適用されないため、容量とパフォーマンスを完全に制御できます。

### カスタムリージョンの設定 \{#custom-region-configuration\}

**name** (必須)

* リージョンを一意に識別する ID
* 英数字、アンダースコア、ピリオド、ハイフンのみ使用できます
* サンドボックスを作成する際にこのリージョンを指定するために使用します

**proxyUrl** (任意)

* このリージョン内のサンドボックスへのトラフィックをルーティングするプロキシサービスの URL
* このリージョン内のランナーがプライベートネットワークにデプロイされている場合に必須です

**sshGatewayUrl** (任意)

* このリージョン内のサンドボックスへの SSH 接続を処理する SSH ゲートウェイの URL
* このリージョン内のランナーがプライベートネットワークにデプロイされている場合に必須です

**snapshotManagerUrl** (任意)

* このリージョン内のスナップショットの保存および取得を処理する Snapshot Manager の URL
* このリージョン内のランナーがプライベートネットワークにデプロイされている場合に必須です

### カスタムリージョンの認証情報 \{#custom-region-credentials\}

カスタムリージョンを作成すると、Daytona は構成したオプションサービス向けに次の認証情報を提供します。

* プロキシサービスが Daytona との認証に使用する API キー
* SSH ゲートウェイサービスが Daytona との認証に使用する API キー
* Daytona がスナップショットマネージャーサービスへアクセスする際に使用する Basic 認証情報

:::note
必要に応じて、これらの認証情報はいつでも再生成できますが、各サービスを更新後の認証情報を用いて再デプロイする必要があります。
:::

## カスタムランナー \{#custom-runners\}

カスタムランナーはあなたの組織によって作成および管理され、独自のランナーマシンを利用し、各カスタムリージョン内でコンピュートリソースを独立してスケールさせることができます。

### カスタムランナーの構成 \{#custom-runner-configuration\}

**name** (必須)

* ランナーの一意の識別子
* 使用できる文字は英字、数字、アンダースコア、ピリオド、ハイフンのみです
* 同じリージョン内に存在する複数のランナーを区別するために使用されます

**regionId** (必須)

* このランナーが割り当てられているリージョンの ID
* 所属する組織が所有するカスタムリージョンである必要があります
* リージョン内のすべてのランナーは、そのリージョンのプロキシおよび SSH ゲートウェイ構成を共有します

### カスタムランナーのトークン \{#custom-runner-token\}

カスタムランナーを作成すると、Daytona はランナーが Daytona への認証に使用するための **トークン** を発行します。

:::note
このトークンは安全な場所に保存してください。後から再度表示することはできません。
:::

### カスタムランナーのインストール \{#installing-the-custom-runner\}

カスタムランナーを登録してセキュアトークンを取得したら、利用しているインフラストラクチャ上に Daytona ランナーアプリケーションをインストールして構成する必要があります。

:::note
ランナーアプリケーションの詳細なインストール手順は、今後のアップデートで提供される予定です。ランナーのインストールに関するサポートが必要な場合は、[support@daytona.io](mailto:support@daytona.io) までお問い合わせください。
:::
