`daytona` コマンドラインツールは、スナップショットの管理や Daytona サンドボックスのライフサイクル管理など、Daytona の中核機能にアクセスするための手段を提供します。インストール手順は[こちら](/docs/getting-started#cli)をご覧ください。

本リファレンスでは、`daytona` コマンドラインツールでサポートされているすべてのコマンドを、その挙動の説明や対応フラグとともに一覧しています。
`daytona` 実行時に `--help` / `-h` フラグを付与すると、各コマンドのドキュメントを参照できます。

## daytona \{#daytona\}

Daytona CLI（コマンドラインインターフェイス）

```shell
daytona [flags]
```

**フラグ**

| ロング         | ショート | 説明                   |
| :---------- | :--- | :------------------- |
| `--help`    |      | Daytona のヘルプを表示します   |
| `--version` | `-v` | Daytona のバージョンを表示します |

| :------- | :--- | :----------- |

<Aside type="note">
  bash シェル環境を使用している場合、オートコンプリート機能をフルに利用するには bash-completion がインストールされていることを確認してください。
  Linux でのインストール: `sudo apt-get install bash-completion`
  macOS でのインストール: `brew install bash-completion`
</Aside>

## daytona create \{#daytona-create\}

新規サンドボックスを作成する

```shell
daytona create [flags]
```

**フラグ**

| ロング                    | ショート | 説明                                              |
| :--------------------- | :--- | :---------------------------------------------- |
| `--auto-archive`       |      | 自動アーカイブ間隔（分）。0 は最大間隔を使用                         |
| `--auto-delete`        |      | 自動削除間隔（分）。負の値は無効、0 は停止時に即時削除                    |
| `--auto-stop`          |      | 自動停止間隔（分）。0 は無効                                 |
| `--class`              |      | サンドボックスのクラス（small、medium、large）                 |
| `--context`            | `-c` | ビルドコンテキストに含めるファイルまたはディレクトリ（複数指定可）               |
| `--cpu`                |      | サンドボックスに割り当てる CPU コア数                           |
| `--disk`               |      | サンドボックスに割り当てるディスク容量（GB）                         |
| `--dockerfile`         | `-f` | サンドボックス用スナップショットの Dockerfile パス                 |
| `--env`                | `-e` | 環境変数（形式: KEY=VALUE）                             |
| `--gpu`                |      | サンドボックスに割り当てる GPU ユニット数                         |
| `--label`              | `-l` | ラベル（形式: KEY=VALUE）                              |
| `--memory`             |      | サンドボックスに割り当てるメモリ（MB）                            |
| `--name`               |      | サンドボックス名                                        |
| `--network-allow-list` |      | サンドボックスで許可する CIDR ネットワークアドレスのカンマ区切りリスト          |
| `--network-block-all`  |      | サンドボックスの全ネットワークアクセスをブロックするか                     |
| `--public`             |      | サンドボックスを公開アクセス可能にする                             |
| `--snapshot`           |      | サンドボックスで使用するスナップショット                            |
| `--target`             |      | ターゲットリージョン（eu、us）                               |
| `--user`               |      | サンドボックスに関連付けるユーザー                               |
| `--volume`             | `-v` | マウントするボリューム（形式: VOLUME&#95;NAME:MOUNT&#95;PATH） |
| `--help`               |      | Daytona のヘルプ                                    |

## daytona snapshot create \{#daytona-snapshot-create\}

スナップショットを作成します

```shell
daytona snapshot create [SNAPSHOT] [flags]
```

**フラグ**

| Long           | Short | 説明                                                                               |
| :------------- | :---- | :------------------------------------------------------------------------------- |
| `--context`    | `-c`  | ビルドコンテキストに含めるファイルまたはディレクトリ（複数回指定可）。指定しない場合は、Dockerfile の COPY/ADD コマンドから自動検出されます |
| `--cpu`        |       | サンドボックスに割り当てる CPU コア数（デフォルト: 1）                                                  |
| `--disk`       |       | サンドボックスに割り当てるディスク容量（GB）（デフォルト: 3）                                                |
| `--dockerfile` | `-f`  | ビルドする Dockerfile のパス                                                             |
| `--entrypoint` | `-e`  | スナップショットのエントリーポイントコマンド                                                           |
| `--image`      | `-i`  | スナップショットのイメージ名                                                                   |
| `--memory`     |       | サンドボックスに割り当てるメモリ（GB）（デフォルト: 1）                                                   |
| `--region`     |       | スナップショットを利用可能にするリージョンの ID（指定がない場合は組織のデフォルトリージョンが使用されます）                          |
| `--help`       |       | Daytona のヘルプ                                                                     |
