`daytona` コマンドラインツールは、Daytona（デイトナ）のコア機能へのアクセスを提供し、スナップショットの管理や Daytona サンドボックスのライフサイクル管理を行えます。インストール手順は[こちら](/docs/ja/getting-started#setting-up-the-daytona-cli)をご覧ください。

本リファレンスでは、`daytona` コマンドラインツールでサポートされているすべてのコマンドを、その動作の説明および対応フラグとともに一覧しています。`daytona` 実行時に `--help` または `-h` フラグを付けることで、各コマンドごとのドキュメントを参照できます。

## daytona

Daytona CLI

```shell
daytona [flags]
```

**フラグ**
| ロング | ショート | 説明 |
| :--- | :---- | :---------- |
| `--help` | | daytona のヘルプを表示します |
| `--version` | `-v` | Daytona のバージョンを表示します |

## daytona autocomplete

シェル環境にオートコンプリート用スクリプトを追加します

```shell
daytona autocomplete [bash|zsh|fish|powershell] [flags]
```

**フラグ**
| Long | Short | 説明 |
| :--- | :---- | :---------- |
| `--help` | | daytona のヘルプを表示 |

<Aside type="note">
  bash シェル環境を使用している場合、完全な補完機能を利用するには
  bash-completion をインストールしていることを確認してください。Linux のインストール: `sudo
        apt-get install bash-completion` macOS のインストール: `brew install
        bash-completion`
</Aside>

## daytona sandbox create

新しいサンドボックスを作成

```shell
daytona sandbox create [flags]
```

**フラグ**
| Long | Short | 説明 |
| :--- | :---- | :---------- |
| `--auto-archive` | | 自動アーカイブの間隔（分）（0 は最大の間隔が使用されます） |
| `--auto-delete` | | 自動削除の間隔（分）（負の値で無効、0 は停止後に即時削除） |
| `--auto-stop` | | 自動停止の間隔（分）（0 は無効） |
| `--class` | | サンドボックスのクラス種別（small、medium、large） |
| `--context` | `-c` | ビルドコンテキストに含めるファイルまたはディレクトリ（複数回指定可） |
| `--cpu` | | サンドボックスに割り当てる CPU コア数 |
| `--disk` | | サンドボックスに割り当てるディスク容量（GB） |
| `--dockerfile` | `-f` | サンドボックスのスナップショット用 Dockerfile のパス |
| `--env` | `-e` | 環境変数（形式: KEY=VALUE） |
| `--gpu` | | サンドボックスに割り当てる GPU ユニット |
| `--label` | `-l` | ラベル（形式: KEY=VALUE） |
| `--memory` | | サンドボックスに割り当てるメモリ（MB） |
| `--public` | | サンドボックスを公開アクセス可能にする |
| `--snapshot` | | サンドボックスに使用するスナップショット |
| `--target` | | ターゲット（リージョン）（eu, us） |
| `--user` | | サンドボックスに関連付けるユーザー |
| `--volume` | `-v` | マウントするボリューム（形式: VOLUME\_NAME:MOUNT\_PATH） |
| `--help` | | daytona のヘルプ |
