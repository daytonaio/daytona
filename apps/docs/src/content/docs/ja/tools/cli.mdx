`daytona`コマンドラインツールは、SnapshotとDaytona Sandboxのライフサイクルの管理を含む、Daytonaのコア機能へのアクセスを提供します。インストール手順は[こちら](/docs/ja/getting-started#setting-up-the-daytona-cli)をクリックしてご確認ください。

このリファレンスは、`daytona`コマンドラインツールでサポートされているすべてのコマンドを、その動作の説明とサポートされているフラグと共に一覧表示しています。
`daytona`を実行する際に`--help`/`-h`フラグを追加することで、コマンドごとにこのドキュメントにアクセスできます。

## daytona

Daytona CLI

```shell
daytona [flags]
```

**フラグ**
| Long | Short | Description |
| :--- | :---- | :---------- |
| `--help` | | daytonaのヘルプ |
| `--version` | `-v` | Daytonaのバージョンを表示 |

## daytona autocomplete

シェル環境用の補完スクリプトを追加します

```shell
daytona autocomplete [bash|zsh|fish|powershell] [flags]
```

**フラグ**
| Long | Short | Description |
| :--- | :---- | :---------- |
| `--help` | | daytona のヘルプ |

<Aside type="note">
  bashシェル環境を使用している場合は、完全な自動補完機能を利用するためにbash-completionがインストールされていることを確認してください。Linuxでのインストール: ```sudo
  apt-get install bash-completion``` macOSでのインストール: ```brew install
  bash-completion```
</Aside>

## daytona sandbox create

新しいサンドボックスを作成する

```shell
daytona sandbox create [flags]
```

**フラグ**
| Long | Short | Description |
| :--- | :---- | :---------- |
| `--auto-archive` | | 自動アーカイブ間隔（分単位）（0は最大間隔が使用されることを意味します） |
| `--auto-delete` | | 自動削除間隔（分単位）（負の値は無効、0は停止時に即座に削除することを意味します） |
| `--auto-stop` | | 自動停止間隔（分単位）（0は無効を意味します） |
| `--class` | | サンドボックスクラスタイプ（small、medium、large） |
| `--context` | `-c` | ビルドコンテキストに含めるファイルまたはディレクトリ（複数回指定可能） |
| `--cpu` | | サンドボックスに割り当てられるCPUコア数 |
| `--disk` | | サンドボックスに割り当てられるディスク容量（GB単位） |
| `--dockerfile` | `-f` | サンドボックススナップショット用のDockerfileへのパス |
| `--env` | `-e` | 環境変数（形式：KEY=VALUE） |
| `--gpu` | | サンドボックスに割り当てられるGPUユニット数 |
| `--label` | `-l` | ラベル（形式：KEY=VALUE） |
| `--memory` | | サンドボックスに割り当てられるメモリ（MB単位） |
| `--public` | | サンドボックスを公開アクセス可能にする |
| `--snapshot` | | サンドボックスに使用するスナップショット |
| `--target` | | ターゲットリージョン（eu、us） |
| `--user` | | サンドボックスに関連付けられるユーザー |
| `--volume` | `-v` | マウントするボリューム（形式：VOLUME_NAME:MOUNT_PATH） |
| `--help` | | daytonaのヘルプ |
