Daytona TypeScript SDK は、Daytona のサンドボックスをプログラムから操作するための強力なインターフェースを提供します。

## インストール \{#installation\}

npm を使用して Daytona の TypeScript SDK をインストールします：

```bash
npm install @daytonaio/sdk
```

または yarn を使用する場合は：

```bash
yarn add @daytonaio/sdk
```

## はじめに \{#getting-started\}

Daytona TypeScript SDK を使い始めるための簡単な例を以下に示します。

```typescript
import { Daytona } from '@daytonaio/sdk'

async function main() {
  // SDKを初期化（デフォルトで環境変数を使用）
  const daytona = new Daytona()

  // 新しいサンドボックスを作成
  const sandbox = await daytona.create({
    language: 'typescript',
    envVars: { NODE_ENV: 'development' },
  })

  // コマンドを実行
  const response = await sandbox.process.executeCommand('echo "Hello, World!"')
  console.log(response.result)
}

main().catch(console.error)
```

## 設定 \{#configuration\}

SDK は、環境変数を使用するか、コンストラクターにオプションを渡すことで設定できます。

```typescript
import { Daytona } from '@daytonaio/sdk';

// 環境変数を使用 (DAYTONA_API_KEY, DAYTONA_API_URL, DAYTONA_TARGET)
const daytona = new Daytona();

// 明示的な設定を使用
const daytona = new Daytona({
  apiKey: 'your-api-key',
  apiUrl: 'https://app.daytona.io/api',
  target: 'us'
});
```

## 複数ランタイム対応 \{#multiple-runtime-support\}

Daytona の TypeScript SDK は、Node.js、Deno、Bun、ブラウザー、サーバーレスプラットフォーム（Cloudflare Workers、AWS Lambda、Azure Functions など）を含む複数の JavaScript ランタイムで動作します。

:::note[ブラウザーおよびフレームワークの設定]
ブラウザー環境や Vite、Next.js などのフレームワークで SDK を使用する場合は、Node のポリフィルを設定する必要があります。セットアップ手順は以下のセクションを参照してください。
:::

### Vite プロジェクトにおける Daytona \{#daytona-in-vite-projects\}

Vite ベースのプロジェクトで Daytona SDK を使用する場合は、互換性を確保するために Node のポリフィルを設定する必要があります。`vite.config.ts` の plugins 配列に次の設定を追加してください:

```typescript
import { nodePolyfills } from 'vite-plugin-node-polyfills'

export default defineConfig({
  plugins: [
    // ... その他のプラグイン
    nodePolyfills({
      globals: { global: true, process: true, Buffer: true },
      overrides: {
        path: 'path-browserify-win32',
      },
    }),
  ],
  // ... 設定の残りの部分
})
```

### Next.js プジェクトでの Daytona \{#daytona-in-nextjs-projects\}

Next.js プロジェクトで Daytona SDK を使用する場合は、利用しているバンドラーに応じて Webpack および Turbopack との互換性を確保するため、Node のポリフィルを設定する必要があります。次の設定を `next.config.ts` ファイルに追加してください。

```typescript
import type { NextConfig } from 'next'
import NodePolyfillPlugin from 'node-polyfill-webpack-plugin'
import { env, nodeless } from 'unenv'

const { alias: turbopackAlias } = env(nodeless, {})

const nextConfig: NextConfig = {
  // Turbopack
  experimental: {
    turbo: {
      resolveAlias: {
        ...turbopackAlias,
      },
    },
  },
  // Webpack
  webpack: (config, { isServer }) => {
    if (!isServer) {
      config.plugins.push(new NodePolyfillPlugin())
    }
    return config
  },
}

export default nextConfig
```
